Cluster Explorer — Welcher Look?

Die Cluster aus Phase P2 (cluster_label, relevanz, treffer_schaetzung) werden als interaktive Visualisierung dargestellt. Welcher Stil passt?

🌌 Galaxy (3D / WebGL)

Dunkler Space-Hintergrund, Cluster als Planeten. Größe = Trefferanzahl, Farbe = Relevanz, Abstand = semantische Distanz (aus pgvector-Embeddings via UMAP). Three.js im Browser, Python berechnet die 3D-Positionen.

🕸 Force-Graph (2D interaktiv)

Heller, klarer Hintergrund. Knoten ziehen sich physikalisch an und ab. Klick auf Cluster zeigt Details. Kanten = semantische Ähnlichkeit. D3.js oder Sigma.js, kein WebGL nötig. Leichter zu embedden.

🫧 Bubble Chart (2D elegant)

Keine Kanten, nur Blasen. Größe = Trefferanzahl, Farbe = Relevanz-Kategorie. Hover zeigt Beschreibung, Click öffnet Detail. Python (Matplotlib/Plotly) exportiert die Positionen, Blade rendert via Chart.js. Am einfachsten zu bauen.
